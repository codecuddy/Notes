C.     Generalizations

                i.         User visits AngularJS app

              ii.         The view presents the app’s data through the use of expressions, filters, and directives.

             iii.         User clicks an element in the view, if the element has a directive it runs the function

             iv.         The function updates the state of the data

              v.         The view automatically changes and displays the updated data.

1.     Note: The page doesn’t need to reload at any point!

2.   Directives

A.     Used app.directive to create a new directive named ‘appInfo’ has 3 parts

                i.         Restrict: specifies how the directive will be used in the review

1.     E means it will be used as a new HTML element

              ii.         Scope: specifies that we’ll pass info into this directive through an attribute. The = tells the directive to look for an attribute with same name

             iii.         templateURL specifies the HTML to use to display data in scope

B.     Usefulness of Directives

                i.         Readability – expressive HTML allows you to understand app’s behavior just by reading HTML

              ii.         Reusability – don’t need to reuse code

C.     Built-In and Custom Directives

                i.         Used to simplify and display static content

              ii.         Used to interact with user (via clicks for example)

D.    Generalizations

                i.         Directives are a powerful way to create self-contained interactive components.

              ii.         AngularJS treats interactivity as a native component of HTML

1.     Unlike jQuery which adds interactivity as a layer on top of HTML
